Part 1: Web and HTML Basics

Worldwide Web

  • Invented in the early 1990's, today the worldwide web has nearly 30 billion web pages
  • The worldwide web is a collection of HTML documents
  • HTML: Hypertext Markup Language and it's the basis of almost every web page
  • URL: Uniform Resource Locators, most commonly used to reference a web page
  • Hyperlinks: Links between pages that make the web, web-like
  • Web applications: Software that runs in a web browser that is created in a browser-supported programming language (such as the combination of JavaScript, HTML and CSS) and relies on a web browser to render the application

Pieces of the Web - Major Concepts

  • Internet: World's largest computer network
  • HTTP: Main protocol of the web that communicates between the internet and the servers
  • Servers: Computers that host the files that make up the web

The Pieces of HTML - Hypertext Markup Language

  • Text content: What you see
  • Markup: What it looks like (tells the web browser how to display the words and images of a web page)
  • References: Access to other documents like images or videos
  • Links: Redirect to other pages

HTML Tag Overview

  • There are opening and closing tags (the name of both tags are always the same)
  • Opening tags are surrounded by a less than and greater than symbol
  • Closing tags are identical to the opening tag with the addition of a slash in front of the tag name
  • An element is comprised of the opening and closing tag with the content in between
  • Leaving off the end tag creates a element that never ends

Basic HTML Tags

  • Anchors: Provide the ability to create a link that when clicked on goes to a web page
  • Line Break: An inline void tag which requires no end tag and makes text appear on multiple lines. Without a line break or a paragraph break, all content in HTML appears on a single line (whitespace)
  • Paragraph: A method to create an invisible box arount the content which can have both height and width

Inline vs. Block Elements

  • Inline Elements: Bold, Italic, Image, Href (to insert a link), Strong, Span, Break
  • Block elements: Used to create an imaginary box around the content - Paragraph, Form, Div (all this does is contain its contents)

Part 2: Web Page Structure Basics

Web Document Structure

  • HTML is the language of the web with the syntax and rules - think of it as the walls of the house. The "words" of HTML are the tags. There are opening tags and closing tags and everything in between is known as the content
  • Each HTML tag creates a series of nested elements that are a tree-like structure know as the DOM tree (Document Object Model tree)
  • CSS is the design/style of the web - think of it as the color of the walls and the carpet
  • Javascript files are the interactive components - think of it as the garage door opener or tv remote

Web Pages are Made of Boxes

  • Web pages can be deconstructed into boxes (even a circle is truly a box)
  • When given a page design, start by determining where the boxes on the page are located from largest to smallest. Worry about the design and placement of the boxes later
  • Div tags are used to define the boxes

Part 3: Adding Design to Web Pages

CSS adds "style" to HTML documents

  • Allows the use of specific syntax and rules to change how elements appear on the page
  • Provides the ability to control the HTML content by telling the HTML how it should look
  • CSS Selector = How to look all of the time (Apply the rules to every h1 element)
  • CSS Class Selectors = How to look some of the time (Apply the rules to certain h1 elements)
  • Including CSS for a web page can be included in the head of the html, with a link in the HTML to a separate CSS file, or written inline with the HTML (not recommended)

Why Avoid Repetition?

  • Saves time by alleviating the need update code in multiple places
  • CSS styles can be updated all at once
  • Helps avoid the likelihood that updates will be missed and mistakes will be made
